// Erkennungslogik & SPL — pro Angriff

01 · Kerberoasting
4769
WAS? TGS-Ticket für SPN mit RC4-Verschlüsselung → Offline-Cracking des Service-Account-Passworts
TRIGGER TicketEncryptionType = 0x17 (RC4) statt 0x12 (AES256)
FILTER ServiceName ≠ krbtgt, ≠ Computer-Accounts (endet auf $)
SCHWELLE ≥ 3 verschiedene SPNs in 10 Min von einer IP = Scanning
// SPL
index=winsec EventCode=4769 TicketEncryptionType="0x17" NOT ServiceName="krbtgt" NOT ServiceName="*$" | bucket _time span=10m | stats dc(ServiceName) as spns by _time,AccountName,IpAddress | where spns >= 3
02 · AS-REP Roasting
4768
WAS? TGT für User ohne Pre-Auth anfordern → AS-REP enthält verschlüsselten Hash → Offline-Crack
TRIGGER PreAuthType = 0 und TicketEncryptionType = 0x17
NORMAL Legitime Logins: PreAuthType = 2 (verschlüsselt mit PW-Hash)
KEIN ACCT Angriff funktioniert anonym — kein Domain-Account nötig
// SPL
index=winsec EventCode=4768 PreAuthType="0" TicketEncryptionType="0x17" Status="0x0" | stats count, values(TargetUserName) as users by IpAddress | sort - count
03 · DCSync
4662
WAS? AD-Replikation imitieren → alle NTLM-Hashes inkl. KRBTGT aus dem DC ziehen (Mimikatz)
TRIGGER Replikations-GUIDs in Properties von einem User-Account (kein DC$)
GUID 1 1131f6aa... → DS-Replication-Get-Changes
GUID 2 1131f6ad... → DS-Replication-Get-Changes-All ← kritisch
FP FILTER SubjectUserName endet auf $ = echter DC → ignorieren
// SPL
index=winsec EventCode=4662 AccessMask="0x100" | where (match(Properties,"1131f6aa") OR match(Properties,"1131f6ad")) AND NOT match(SubjectUserName,"\$$") | table _time,SubjectUserName,ComputerName
04 · Golden Ticket
4624 + 4672
WAS? KRBTGT-Hash → beliebige TGTs fälschen → als jeder User, für bis zu 10 Jahre
TRIGGER 4624 LogonType=3 Kerberos ohne vorheriges 4768 vom gleichen Host
EXTRA 4672 mit SeTcbPrivilege = Mimikatz-Fingerabdruck
ACHTUNG Gilt auch nach PW-Reset von Admin — KRBTGT muss 2× zurückgesetzt werden
// SPL — Anomalie via Privileges
index=winsec EventCode=4672 | where match(PrivilegeList, "(?i)SeTcbPrivilege") | table _time,SubjectUserName, IpAddress,PrivilegeList
05 · Pass-the-Ticket
4769 ohne 4768
WAS? Gestohlenes .kirbi-Ticket in andere Session injizieren — kein Hash nötig
TRIGGER 4769 (TGS-Request) ohne zeitlich passendes 4768 (TGT) vom gleichen AccountName
PtH vs PtT Pass-the-Hash → NTLM (4776). Pass-the-Ticket → Kerberos (4769). Einfach am Auth-Paket erkennbar.
// SPL — TGS ohne TGT in letzter Stunde
index=winsec EventCode=4769 | eval t1=_time | join type=left AccountName [ search index=winsec EventCode=4768 | eval t2=_time | table AccountName,t2 ] | where isnull(t2) OR abs(t1-t2) > 3600 | table _time,AccountName, ServiceName,IpAddress

// Master Query — alle 5 Angriffe kombiniert

index=winsec ( (EventCode=4769 TicketEncryptionType="0x17" NOT ServiceName="krbtgt" NOT ServiceName="*$") ← Kerberoasting OR (EventCode=4768 PreAuthType="0") ← AS-REP Roasting OR (EventCode=4662 AccessMask="0x100") ← DCSync OR (EventCode=4672 match(PrivilegeList,"SeTcbPrivilege")) ← Golden Ticket OR EventCode=4728) ← Persistence: DA-Gruppe | eval attack = case( EventCode=4769, "Kerberoasting/PtT", EventCode=4768, "AS-REP-Roasting", EventCode=4662, "DCSync", EventCode=4672, "Golden-Ticket", EventCode=4728, "DA-Persistence") | eval sev = case( EventCode=4662, "CRITICAL", EventCode=4728, "CRITICAL", 1=1, "HIGH") | table _time, sev, attack, EventCode, TargetUserName, IpAddress, ComputerName | sort - _time
