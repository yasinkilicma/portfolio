Was ist BOTS v1?

Boss of the SOC (BOTS) v1 ist ein öffentlich verfügbarer Splunk-Datensatz, der einen realen Cerber-Ransomware-Angriff aus dem Jahr 2016 dokumentiert. Er enthält 955.807 Events aus sechs verschiedenen Datenquellen und deckt die vollständige Angriffs-Kill-Chain ab — von der ersten Reconnaissance bis zur Ransomware-Ausführung und Persistence-Mechanismen.

Der Datensatz wird von Splunk für SOC-Training und CTF-Wettbewerbe verwendet und bildet die Grundlage für zahlreiche reale Analyst-Übungen.

→

Lab-Setup: Splunk läuft als Docker-Container, provisioniert mit Terraform. Der BOTS v1 Datensatz wurde als Splunk App importiert. Die komplette Lab-Architektur ist im Lab-Setup Writeup beschrieben.

Die vollständige Kill Chain

01
// Reconnaissance — Network Scanning
Nessus Reconnaissance & Port Scanning
suricata Nessus Scanner
Der Angreifer führt einen umfassenden Nessus-Scan gegen das Zielnetzwerk durch. Suricata erkennt den Scanner anhand des charakteristischen Scan-Musters — viele Ports in kurzer Zeit, SYN-Pakete ohne vollständigen Handshake. Die gescannten Ports geben Aufschluss über die geplante Angriffsstrategie.
INFO
02
// Initial Access — Web Exploitation
Suricata IDS — Web-Angriffe & Cerber
1.360 Alarme Acunetix CVE-2014-6271 XSS / SQLi
Der Acunetix Web-Scanner führt automatisierte Angriffe durch: XSS, SQL-Injection und Shellshock (CVE-2014-6271) gegen einen verwundbaren Apache-Webserver. 1.360 Suricata-Alarme aus 125.584 IDS-Events. Shellshock ermöglicht Remote Code Execution über einen präparierten HTTP-Header — der Einstiegspunkt der Cerber-Infektion.
HIGH
03
// Lateral Movement — SMB Propagation
SMB Lateral Movement — Cerber Ausbreitung
stream:smb 151.568 Events 192.168.250.100
Nach der initialen Kompromittierung verbreitet sich Cerber über SMB im internen Netzwerk. Netzwerk-Forensik auf 151.568 SMB-Events zeigt den Ausbreitungspfad: 192.168.250.100 → 192.168.250.20. SMB-Write-Operationen auf ungewöhnliche Pfade und auffällige Dateinamen sind die entscheidenden Indikatoren.
HIGH
04
// Execution — Process Masquerading
Sysmon Process Creation — Masquerading Detection
Sysmon EventID 1 osk.exe 767 Events
Sysmon EventID 1 (Process Creation) deckt auf: Die Ransomware tarnt sich als osk.exe (On-Screen Keyboard) — ein legitimer Windows-Prozess. Hash-Validierung beweist das Masquerading: der SHA256-Hash stimmt nicht mit dem echten Microsoft-Binary überein. Ohne Sysmon wäre dieser Prozess im normalen Windows Event Log unsichtbar.
CRITICAL
05
// Persistence — Registry Run Key
Registry Persistence — Run Key Analysis
WinRegistry Run Key 74.720 Events
Cerber sichert seine Persistence über den Windows Registry Run Key: osk.exe und internat.exe werden als Auto-Start-Einträge registriert. 74.720 Registry-Events analysiert — der entscheidende Eintrag findet sich unter HKCU\Software\Microsoft\Windows\CurrentVersion\Run. Jeder Reboot startet die Ransomware neu.
CRITICAL
06
// C2 & Exfiltration — DNS Anomaly
FortiGate Firewall — C2 & DNS-Analyse
fortigate_utm C2 192.168.250.20 DNS Exfil
FortiGate-Logs zeigen ausgehende C2-Kommunikation: 192.168.250.20 → 8.8.8.8 über ungewöhnliche Ports. DNS-Exfiltrations-Muster erkennbar anhand überlanger Hostnamen und hoher Query-Frequenz. Die Firewall-Analyse zeigt auch, wo die Perimeter-Defence versagte — nur 20% Effektivität.
HIGH
