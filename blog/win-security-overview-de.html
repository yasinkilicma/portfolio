Die Ausgangslage

Für diese Analyse habe ich ein Python-Skript entwickelt, das 175 realistische Windows Security Events erzeugt und per HTTP Event Collector (HEC) direkt in einen lokalen Splunk-Container injiziert. Die Events simulieren eine vollständige Angriffskampagne in einer fiktiven Windows-Domain corp.local mit vier Hosts:

LAB ENVIRONMENT corp.local — fiktive Windows-Domain
DC01.corp.local ← Domain Controller (Hauptziel der Angreifer) WS01.corp.local ← Workstation 1 (Pass-the-Hash Ausgangspunkt) WS02.corp.local ← Workstation 2 (Macro Malware Opfer) SRV-FILE01.corp.local ← File Server (normale Aktivität) Benutzer: jsmith, amüller, tkurt, sdemir, fbauer Angreifer-IP: 185.220.101.5 ← Tor-Exit-Node

Die vollständige Kill Chain

Die fünf Szenarien bilden zusammen eine realistische Angriffskampagne — von der ersten Erkundung bis zur dauerhaften Backdoor:

01
// Initial Access — Credential Attack
Brute-Force-Angriff gegen jsmith
4625 × 10 4740
Tor-Exit-Node 185.220.101.5 versucht 10-mal das Passwort von jsmith auf DC01 — alle mit Status 0xC000006A (wrong password). Nach dem 10. Versuch greift EventID 4740: automatische Kontosperrung.
HIGH
02
// Lateral Movement — Credential Theft
Pass-the-Hash & Lateral Movement
4776 4648 4624 Type 3 4672
Mit dem NTLM-Hash des Administrator-Kontos (aus dem Speicher von lsass.exe via Mimikatz) authentifiziert sich jsmith als Administrator auf DC01 — ohne das Klartextpasswort zu kennen. NTLM statt Kerberos bei einem DC-Login ist der entscheidende Indikator.
HIGH
03
// Privilege Escalation — Account Manipulation
Privilege Escalation zu Domain Admins
4720 4732 4728
jsmith legt das Backdoor-Konto hacker_svc an (4720), fügt es zur lokalen Administrators-Gruppe hinzu (4732) und schließlich zu Domain Admins (4728) — alles in 2 Minuten. Die Domain ist vollständig kompromittiert.
CRITICAL
04
// Persistence — Backdoor Installation
Persistence — Scheduled Task & Malicious Service
4698 7045
Als Domain Admin installiert der Angreifer zwei Backdoors auf DC01: einen Scheduled Task (\Microsoft\Windows\WindowsUpdate\Scheduled Start) mit encoded PowerShell-Payload und einen Auto-Start-Service (svch0st.exe — typosquatting svchost) aus C:\Windows\Temp\.
CRITICAL
05
// Initial Access (parallel) — Phishing
Macro Malware — WINWORD → cmd → PowerShell → net.exe
4688 × 4
Parallel zur DC-Kompromittierung öffnet fbauer auf WS02 eine Phishing-Mail-Anlage (Rechnung_2026_06.docm). Das Makro startet in 3 Minuten: WINWORD → cmd → powershell -EncodedCommand → net user backdoor P@ssw0rd123! /add. EventID 4688 zeichnet jeden Schritt auf.
CRITICAL
