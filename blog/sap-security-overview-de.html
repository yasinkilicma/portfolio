Was ist das SAP Security Audit Log?

Das SAP Security Audit Log (SAL) ist die wichtigste Sicherheitsprotokollquelle in SAP-Systemen. Es wird über Transaktion SM20 ausgelesen und protokolliert sicherheitsrelevante Ereignisse: Anmeldungen, Anmeldefehler, Berechtigungsprüfungen, RFC-Zugriffe und Tabellenoperationen.

Für diese Analyse habe ich ein Python-Skript entwickelt, das 128 realistische SAP Audit Log Events über 5 Angriffsszenarien generiert und per Splunk HEC in einen lokalen Splunk-Container injiziert. Anschließend wurde ein vollständiges Security Monitoring Dashboard gebaut.

→

Relevanz für die Praxis: SAP-Systeme enthalten die kritischsten Geschäftsdaten eines Unternehmens — Finanzdaten, Personalinformationen, Lieferantenstammdaten. Angriffe auf SAP bleiben oft lange unentdeckt, weil viele SOC-Teams keine SAP-spezifischen Detection Rules haben. Der SAP Security Audit Log ist der erste Schritt zur Sichtbarkeit.

Die 5 Angriffsszenarien

01
// Credential Attack — External
SAP Brute-Force-Angriff
AU2 AUI AU4
Externe IP 185.220.101.5 versucht 12-mal mit Default-Benutzernamen (SAP*, DDIC, EARLYWATCH) auf das Produktivsystem zuzugreifen. AU2-Events häufen sich, schließlich greift AU4 — automatische Kontosperrung. Klassisches Muster für automatisierte SAP-Angriffs-Tools.
HIGH
02
// Insider Threat — Compromised Admin
Kompromittiertes SAP-Admin-Konto
AU1 DU9
SAPABAP1 meldet sich um 02:30 Uhr von einer externen IP an — außerhalb jeder normaler Geschäftszeit. Anschließend werden kritische Transaktionen ausgeführt: SE38 (ABAP-Editor), SU01 (Benutzerverwaltung), PFCG (Rollenpflege). DU9-Events dokumentieren jeden Eingriff in die Benutzeradministration.
CRITICAL
03
// Privilege Escalation — AUB Storm
SAP Privilege Escalation — AUB Storm
AUB × 15
Benutzer TKURT versucht 15-mal auf privilegierte Transaktionen zuzugreifen: PFCG, SE11, SM30, SM59, SU01. Jeder Versuch erzeugt einen AUB-Event (Authorization Check Failed). Ein gehäuftes AUB-Muster von einem einzelnen Benutzer auf kritische Transaktionen ist ein klares Privilege-Escalation-Signal.
HIGH
04
// External Attack — RFC Exploitation
Externer SAP RFC-Angriff
AU1 AU2
185.220.101.5 testet über RFC-Verbindungen typische Exploit-Funktionsbausteine: RFC_READ_TABLE (Tabellen auslesen), RFCPING (Erreichbarkeit prüfen), SAPMSSY1 (ältere Systemfunktion). Das Muster — mehrere RFC-Funktionen von einer externen IP mit Default-Credentials — ist das klassische Fingerprint automatisierter SAP-Angriffs-Tools wie bizploit oder ERPScan.
HIGH
05
// Data Exfiltration — Mass Table Read
SAP Datenexfiltration
EU1
SDEMIR liest in einer einzigen Session sechs hochsensible Tabellen aus: USR02 (Passwort-Hashes), USR04 (Berechtigungen), AGR_USERS (Rollenzuordnungen), RFCDES (RFC-Verbindungen), ABAP_SOURCE (Quellcode), T000 (Mandanten). Über 20.000 Datensätze in Sekunden — EU1-Events als Exfiltrations-Indikator.
CRITICAL
