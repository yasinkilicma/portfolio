Project Overview

Most network audits involve running multiple disconnected tools and stitching results together manually. I wanted a single Python tool that scans a target, identifies running services, checks them against CIS hardening guidelines, and generates a professional report — all in one command.

audit.py (Orchestrator) │ ├── PortScanner → TCP-Connect scan, CIDR support, parallel threads ├── ServiceChecker → Banner grabbing, SSH version, HTTP header analysis ├── CISBenchmark → 9 controls: Telnet, SMB, RDP, DB ports, SSH, Redis… └── ReportGenerator → Text (ANSI colour), JSON (SIEM), HTML (Bootstrap 5)

The Port Scanner

The scanner uses Python's socket module with a TCP-Connect scan. Unlike SYN scans, this doesn't require root privileges and works on any platform. A ThreadPoolExecutor runs up to 100 port checks in parallel, reducing scan time dramatically.

Service Analysis & Banner Grabbing

After discovering open ports, the ServiceChecker attempts to grab the service banner. Many services (FTP, SSH, SMTP) automatically send a welcome message containing their name and version number — this is called a banner and it's a goldmine of information for both defenders and attackers.

ℹ️

Example: Connecting to port 22 and receiving SSH-2.0-OpenSSH_7.2p2 immediately reveals an outdated SSH version (7.2p2 was released in 2016 and has known CVEs).

Report Generator

The same findings are rendered in three formats depending on the use case:

🖥️
Text (Terminal)

ANSI-coloured output with severity-coded findings. CRITICAL in red, HIGH in orange, MEDIUM in yellow. Perfect for quick command-line assessments.

📄
JSON (SIEM Integration)

Structured output suitable for ingestion into Splunk, Elastic, or any SIEM platform. Timestamped, fully machine-readable.

🌐
HTML (Bootstrap 5)

Self-contained report with a summary dashboard (open ports, finding count, overall risk level) and colour-coded finding cards.

Key Takeaways

01
Threading Makes the Difference

Scanning 1024 ports sequentially takes ~17 minutes at 1s timeout. With 100 parallel threads, the same scan completes in under 11 seconds.

02
Banners Are Intelligence

A single banner response often reveals the exact software version, enabling targeted CVE lookups without any additional scanning.

03
CIS = Concrete Guidance

Mapping findings to CIS IDs transforms raw port data into actionable remediation tasks that can be tracked and reported to management.

04
Multiple Output Formats

The same scan results flow into three different renderers — terminal for operators, JSON for automation, HTML for stakeholders.
